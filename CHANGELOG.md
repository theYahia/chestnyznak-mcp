# Changelog

## 1.2.0

API-корректность, надёжность и безопасность. Эндпоинты выверены по официальной
документации ЦРПТ и независимым open-source клиентам (публичный API геоблокируется
с не-РФ IP, поэтому живой ответ публичной части подтверждён частично, авторизованная —
не проверена против живого токена из-за требования УКЭП).

### Корректность API
- Публичная проверка теперь шлёт обязательный параметр `codeType` (`datamatrix`/`qr`/`ean13`)
  и использует канонический `GET /mobile/check?code=&codeType=`.
- Результат проверки читается из реального поля `checkResult` (раньше — несуществующее
  `isValid`, из-за чего `valid` всегда был `false`).
- Детали (`status`/`producerName`/`ownerName`/`ownerInn`) извлекаются из вложенного
  объекта группы товара `<category>Data` с fallback на корень.
- Удалены поля, которых нет в публичном API (`statusText`, `productGroupName`,
  `producerInn`, `brand`).
- True API: база исправлена `v4`→`v3`, убран задвоенный сегмент `/true-api/true-api/`,
  `cis/info`→`cises/info` (POST с массивом). `search_products` переведён на
  `POST /api/v4/true-api/product/info` по GTIN.
- Документирована реальная модель авторизации True API (cert-challenge УКЭП, токен ~10ч).

### Надёжность
- Retry теперь срабатывает и на сетевые ошибки (`TypeError`), не только на таймаут.
- Защита от не-JSON/пустых ответов; дружелюбное сообщение о таймауте.
- `check_batch`: ограничение параллелизма (5 одновременных запросов) против 429.
- Единый контракт ошибок: инструменты бросают исключение, MCP-обёртка возвращает
  `isError: true` (раньше отсутствие токена возвращалось как «успех»).
- Тримминг входных кодов.

### Безопасность
- SSRF-защита: `CHESTNYZNAK_BASE_URL` валидируется (только `https`, `*.crpt.ru`/`*.crptech.ru`).
- HTTP-режим: bind на `127.0.0.1` по умолчанию, защита от DNS-rebinding (`Host`-валидация),
  stateless-транспорт per-request (исправлен мульти-клиент), try/catch вокруг обработки,
  CORS-preflight. Открыть наружу — `--host=0.0.0.0`.
- Убран `rawResponse`-дамп апстрим-ответа в вывод LLM.

### Качество / DX
- `outputSchema` + `structuredContent` для всех инструментов.
- Версия сервера читается из `package.json` (без дрейфа).
- Тесты: retry/backoff/timeout, конкуррентность batch, SSRF-guard, success-path
  авторизованных инструментов (23 теста).
- CI: матрица Node 18/20/22, порядок `typecheck → build → test`, coverage; добавлен
  `publish.yml` с npm provenance.
- Исправлен испорченный `package-lock.json` (git-конфликтные маркеры, версии 1.1.0↔1.1.1).
- `engines.node` поднят до `>=18.18.0` (требование зависимостей).
